# Amica VDF Generator

Этот инструмент предназначен для генерации файлов Amica .VDF (XML) путем объединения XML-шаблонов, данных из JSON и обновления информации о CSV-источниках данных.

## Возможности

- Подстановка статических данных из JSON в VDF шаблоны.
- Поддержка преобразований данных (даты, регулярные выражения).
- Автоматическое вычисление MD5 хеша для файлов данных (CSV/TXT).
- Логирование с ротацией файлов.
- Поддержка CDATA для текстовых блоков в VDF.

## Использование

```bash
python amica_generator.py --template <путь_к_шаблону> --csv <путь_к_csv> --json <путь_к_json> --mapping <путь_к_mapping.json> --output <путь_к_результату>
```

### Аргументы

- `--template`: Путь к базовому VDF шаблону.
- `--csv`: Путь к новому CSV/TXT файлу данных. Его путь и MD5 будут записаны в VDF.
- `--json`: Путь к файлу со статическими данными (JSON).
- `--mapping`: Путь к файлу маппинга (по умолчанию `mapping.json`).
- `--output`: Путь для сохранения сгенерированного VDF файла.

## Формат Mapping JSON

Файл маппинга определяет, какие ключи из JSON данных должны подставляться вместо каких текстовых меток в VDF шаблоне.

### 1. Простой маппинг

Используется для прямой замены текста.

```json
{
  "Product_article": "Product_article_placeholder"
}
```
*В этом примере значение ключа `Product_article` из JSON заменит все вхождения `Product_article_placeholder` в шаблоне.*

### 2. Маппинг с преобразованиями

Позволяет изменять данные перед вставкой в шаблон. Преобразования применяются последовательно.

```json
{
  "Batch_date_production": {
    "placeholder": "Date_Placeholder",
    "transform": [
      {
        "type": "strptime",
        "format": "%Y-%m-%d"
      },
      {
        "type": "strftime",
        "format": "%m.%Y"
      }
    ]
  },
  "Product_article": {
    "placeholder": "Article_Placeholder",
    "transform": [
      {
        "type": "regex",
        "pattern": "^[A-Z]+-",
        "replacement": ""
      }
    ]
  }
}
```

#### Доступные типы преобразований:

1.  **`strptime`**: Парсит строку в объект даты и времени.
    - `format`: Формат входящей строки (например, `%Y-%m-%d`).
2.  **`strftime`**: Преобразует объект даты в строку нужного формата.
    - `format`: Желаемый формат исходящей строки (например, `%d.%m.%Y`).
3.  **`regex`**: Выполняет замену по регулярному выражению (`re.sub`).
    - `pattern`: Регулярное выражение для поиска.
    - `replacement`: Строка для замены.

## Логирование

Все операции и ошибки записываются в файл `amica_generator.log`.
- Лог-файл автоматически ротируется при достижении размера 1 МБ.
- Хранится до 5 предыдущих версий логов.

## Ошибки

Скрипт завершится с ошибкой (exception), если:
- Указанный в маппинге ключ отсутствует в JSON файле.
- Одно из преобразований не удалось выполнить (например, неверный формат даты).
- Файл шаблона или данных не найден.
